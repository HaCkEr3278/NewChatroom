Dockerfile for Node.js / Bun Project

Dockerfile Overview

        FROM bun:latest

        ENV NODE_ENV=production

        WORKDIR /public

        COPY ["bun.lockb", "package.json", "./"]

        RUN bun install

        COPY . .

        CMD ["bun", "run", "start"]

Explanation of the Dockerfile:

  • FROM bun:latest: Uses the official Bun image as the base, optimized for Bun's JavaScript runtime.
  • ENV NODE_ENV=production: Sets the environment variable for production, optimizing for performance in live environments.
  • WORKDIR /public: Sets `/public` as the working directory inside the container where your application files will live.
  • COPY ["bun.lockb", "package.json", "./"]: Copies the Bun lockfile and package manifest to ensure consistent dependency management.
  • RUN bun install: Installs dependencies defined in `bun.lockb` and `package.json` using Bun.
  • COPY . .: Copies the rest of your application code into the container.
  • CMD ["bun", "run", "start"]: Defines the default command to run the application using Bun.

Improvements & Best Practices

Enhance your Dockerfile's performance, security, and reliability:

  • Optimizing Docker Caching: Separate dependency installation from application copy to cache layers efficiently.
  • Ensure sensitive files (like `.env` and `.git`) are excluded by adding a .dockerignore file to your project.
  • For added reliability, consider adding a health check to monitor the status of the container and automatically restart it if needed.
